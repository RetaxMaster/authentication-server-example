Para este caso donde únicamente manejaremos el flujo de Password Grant Tokens, no necesitamos que el usuario pueda loguearse a nuestro servicio de autenticación porque en este flujo únicamente recibiremos el usuario y la contraseña del usuario. Este flujo es únicamente para aplicaciónes de alta confianza.

Sin embargo, necesitamos poder registrar usuarios así que tenemos que levantar una ruta manual de registro de usuarios, gracias al archivo config/auth.php Laravel sabe que debe usar al modelo (y la tabla) de User para hacer la revisión y autenticación del usuario (revisión de email y contraseña, generación de tokens, etc.), es por eso que nosotros tenemos que levantar una ruta de registro de usuarios que trabaje con ese modelo, de esta forma podemos registrar usuarios desde nuestra ruta, y autenticar usuarios y generar tokens con las rutas de OAuth.
